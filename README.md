# javbus-api_magnetic-selection-script
JAV 磁力优选脚本 (青龙面板)
基于青龙面板的番号磁力链接自动抓取与智能优选工具。通过 javbus-api 获取磁力数据，支持缓存持久化、中文字幕优先、广告水印过滤、文件体积优选，并可将结果导出为 Excel。

调用的api为：ovnrain / javbus-api项目，需docker自行部署。https://github.com/ovnrain/javbus-api

✨ 主要特性
双模式运行：可通过环境变量切换「优选模式」和「缓存补全模式」。

智能优选（优选模式）：

中文字幕优先：优先选取包含中文字幕的磁力。

广告水印过滤：内置三级过滤名单，自动剔除含论坛水印、破解声明、发布者签名、广告域名的磁力。

体积优选：自动选择文件大小最接近目标体积（默认 2.5GB）的磁力，适合手机等移动端观看。

本地缓存：所有 API 数据会存入 CSV，大幅减少重复请求，支持断点续传。

失败计数：对连续失败的番号自动记录，后续运行跳过，避免无效重试。

通知推送：自动适配青龙面板的 sendNotify，任务完成后推送统计结果。

输出 Excel：优选模式下生成 .xlsx 文件，包含成功记录和失败番号两个工作表。

多任务支持：可一次处理多个输入文件，每个文件作为独立任务统计。

🚀 快速开始
1. 环境要求
青龙面板（或任何支持 Python 脚本的定时任务平台）

Python 3.x

需要安装的依赖（青龙面板通常已内置）：

requests

openpyxl（仅在优选模式下需要）

2. 部署脚本
将脚本文件（如 javbus-api_Magnetic-selection-script.py）放入青龙面板的 scripts 目录。

在青龙面板中创建任务，设置执行命令：

bash
task javbus-api_Magnetic-selection-script.py
根据你的需要，在青龙面板的环境变量中设置：

变量名：ENABLE_OPTIMIZE，值设为 true 开启优选模式，否则默认为缓存补全模式。

3. 准备输入文件
脚本会在与脚本同名的目录下自动创建 input、cache、output 三个文件夹。
你只需要将包含番号的 .txt 文件放入 input 目录中。

输入文件格式示例：

text：

任务名（可中文，用于输出 Excel 文件名）

ABC-123

DEF-456

GHI-789

第一行为任务名称，后续每行为一个番号（大小写不敏感，但建议保持标准格式如 SSIS-001）。

文件名必须包含字母 id（例如 my_movie_ids.txt），否则不会被识别。

4. 运行
脚本会依次处理 input 目录下的每个文件，处理完毕后会通过青龙通知（或日志）输出结果。

⚙️ 配置说明
基本配置
变量	默认值	说明
BASE_URL	http://localhost:8922/api	本地磁力 API 地址
SIZE_MIN	1.1 * 1024**3 (1.1 GB)	优选时的最小文件体积
SIZE_MAX	5.2 * 1024**3 (5.2 GB)	优选时的最大文件体积
TARGET	2.5 * 1024**3 (2.5 GB)	优选时的目标体积，脚本会选最接近的值
💡 为什么默认 2.5 GB？
对于 1080P 主流码率的影片，2.5 GB 是画质与体积的黄金平衡点，既能在手机上获得清晰画质，又不会占用过多存储空间。

环境变量控制
变量名	值	效果
ENABLE_OPTIMIZE	true	开启优选模式（过滤+选最佳）
ENABLE_OPTIMIZE	其他/不设置	仅补全缓存，不做优选

🧹 广告过滤逻辑
脚本采用三级过滤 + 优先顺序机制：

中文字幕优先：若存在字幕资源，则仅在此范围内筛选，否则全部候选。

AD_L1 强干扰：包含以下任一关键词的磁力直接丢弃：

论坛水印：@sis001、t66y、第一會所 等

发布者签名：店長推薦、牛大力、tanw或yk 等

破解/流出：破解、uncen、無修正、leaked 等

广告域名：.cc、.tv、.xyz、.top

AD_L2 软惩罚：不会直接丢弃，但会在体积计算时加上惩罚值，使其在优选排序中靠后：

AD_L2_HEAVY（+5GB 惩罚）：老旧格式 .rmvb、.wmv、.avi，压缩包，标清标签 sd

AD_L2_LIGHT（+2GB 惩罚）：标题党画质标签 4k、60fps、x264 等

体积优选：在剩余的干净资源中，选择经过惩罚调整后体积最接近 TARGET 的那一条。

兜底机制：如果某个番号的所有磁力都被 L1 过滤干净，脚本会自动回退到过滤前的列表，确保至少输出一条结果，不会遗漏番号。

📊 输出说明
缓存模式：仅输出日志，不生成 Excel；所有新增磁力追加到 CSV 缓存。

优选模式：

生成 Excel 文件 output/任务名.xlsx，包含：

工作表“磁力链接”：所有成功优选的记录（含番号、磁力ID、标题、大小、字幕等字段）

工作表“失败番号”：未能获取到磁力的番号及错误原因

通知中会包含本次任务的统计信息：缓存命中数、API 查询数、成功数、中文字幕数、失败数等。

🔧 进阶自定义
调整体积偏好：修改脚本开头的 SIZE_MIN、SIZE_MAX、TARGET。

修改过滤关键词：直接编辑 AD_L1、AD_L2_HEAVY、AD_L2_LIGHT 三个列表，增删词条即可。

更改缓存文件名：修改 CACHE_FILE 变量。

调整重试次数：修改 MAX_RETRIES 和 RETRY_DELAY。

❓ 常见问题
Q: 为什么有些番号获取不到磁力？
A: 可能原因：番号在 API 源中不存在、API 服务未启动、网络问题或已超过失败重试上限。可检查 failed_counts.json 和日志。

Q: 优选模式下全部失败怎么办？
A: 查看 output 中的 Excel 失败工作表，结合 failed_counts.json 分析原因。可尝试手动清理该番号的失败计数以重新尝试。

Q: 我不需要中文字幕优先，想改成按体积绝对优先？
A: 修改 select_best 函数，注释掉字幕优先那段代码即可。

Q: 如何更换 API 源？
A: 修改 BASE_URL 和 api_detail、api_magnets 函数中的请求逻辑，使其适配你的新 API。
